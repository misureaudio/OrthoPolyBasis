• # Chebyshev Module Description

  The chebyshev directory contains a Python package for Chebyshev polynomials — a family of orthogonal polynomials
  widely used in numerical analysis, approximation theory, signal processing, and spectral methods.

  ———

  ## Module Overview

  ### 1. core.py — Core Numerical Engine (Pure Python)

  The foundational module implementing numerically stable Chebyshev operations without external dependencies.

  | Component | Purpose |
  |-----------|---------|
  | ChebyshevGenerator class | Main generator with caching; provides derivative/integral series and Clenshaw evaluation
  |
  | get_derivative_series(n) | Returns coefficients of T'_n in the Chebyshev basis (stable, avoids monomials) |
  | get_integral_series(n) | Returns coefficients of ∫T_n dx in the Chebyshev basis |
  | evaluate_series(x, coeffs) | Evaluates Σa_i·T_i(x) using Clenshaw's algorithm — the numerically stable standard for
  Chebyshev series |
  | chebyshev_derivative_stable(n, x) | Convenience function: computes T'_n(x) directly |
  | chebyshev_integral_stable(n, x) | Convenience function: computes ∫T_n(t)dt from 0 to x |

  Key design: Avoids converting to monomial basis (x^n), which is numerically unstable for large n. Works entirely in
  the Chebyshev basis.

  ———

  ### 2. sympy_integration.py — Symbolic Mathematics Interface

  Provides symbolic manipulation of Chebyshev polynomials using SymPy.

  | Component | Purpose |
  |-----------|---------|
  | SympyChebyshevGenerator class | Generates symbolic Chebyshev expressions with caching |
  | get_polynomial(n) | Returns T_n(x) as a SymPy expression (uses sympy.polys.orthopolys.chebyshevt_poly) |
  | get_polynomial_manual(n) | Manual recurrence-based generation: T_k = 2x·T_{k-1} - T_{k-2} |
  | get_derivative(n) | Symbolic derivative d/dx[T_n(x)] |
  | get_integral(n) | Symbolic indefinite integral ∫T_n(x)dx |
  | evaluate(x_val, n) | Evaluates T_n at a numeric point (returns float) |
  | generate_basis(max_n) | Returns list [T_0, T_1, ..., T_{max_n}] as SymPy expressions |

  Use case: Exact symbolic computation, algebraic manipulation, analytical derivatives/integrals.

  ———

  ### 3. numpy_integration.py — Numerical Array Interface

  High-performance numerical evaluation using NumPy's polynomial machinery.

  | Component | Purpose |
  |-----------|---------|
  | NumpyChebyshevGenerator class | Wraps NumPy's Chebyshev module for batch operations |
  | get_coefficients(n) | Returns T_n coefficients as numpy array (ascending order: constant first) |
  | evaluate(x, n) | Evaluates T_n(x) at a point or array |
  | evaluate_batch(xs, n) | Vectorized evaluation over multiple points |
  | generate_basis(max_n) | Returns list of coefficient arrays for T_0 through T_{max_n} |

  Use case: Fast numerical computation, vectorized operations, integration with scientific Python stack.

  ———

  ### 4. mpmath_integration.py — Arbitrary Precision Interface

  High-precision arithmetic using mpmath (supports hundreds of decimal places).

  | Component | Purpose |
  |-----------|---------|
  | MpmathChebyshevGenerator class | High-precision Chebyshev operations with configurable precision |
  | get_polynomial(n) | Returns callable T_n that respects caller's precision context |
  | get_derivative(n) | Numerical differentiation at arbitrary precision |
  | get_integral(n) | Numerical integration from 0 to x at arbitrary precision |
  | evaluate(x, n, dps) | Evaluates T_n(x) with specified decimal places (default: 50) |

  Key feature: Uses mp.workdps() context manager — doesn't pollute global precision state.

  Use case: High-precision requirements, numerical verification, research-grade computations.

  ———

  ### 5. tests.py — Test Suite

  Comprehensive tests for the core module:

  - Known value verification (T_n(cos θ) = cos(nθ))
  - Derivative and integral coefficient validation
  - Batch evaluation correctness
  - Numerical stability at boundary points x = ±1, 0
  - Edge cases (n = 0, n = 1)

  ———

  ### 6. __init__.py — Package Interface

  Exports public API:
